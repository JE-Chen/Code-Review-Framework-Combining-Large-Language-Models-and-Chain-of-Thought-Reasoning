# 論文撰寫通用教訓

本文件僅收錄跨研究主題與跨專案適用的原則。個別研究的實驗條件、模型、資料規模、
進度、結果數字、服務設定與檔案位置，不得寫入通用代理規則。

## 證據與主張

1. 不捏造數字、資料來源、研究問題、比較條件、引用或結論。
2. 每項事實主張都應能回溯至原始資料、正式文獻、程式實作或可重現的分析結果。
3. 清楚區分已完成結果、合理推論、研究限制與後續工作，不得把設計目標寫成已驗證成果。
4. 跨資料集、跨評估者或跨實驗批次的數值，除非比較設計允許，否則只作描述性對照。
5. 未取得充分證據時，縮小主張範圍，不以肯定語氣填補資料缺口。

## 指標與統計

1. 首次使用指標時說明分子、分母、計數單位、分數範圍及數值方向。
2. 名稱相同的指標若計算方式不同，必須明確區分，不得直接並列比較。
3. 相同樣本在不同條件下的結果應視為成對資料，分析方法須符合資料結構。
4. 同時比較多個指標時，應說明多重比較、估計不確定性及統計檢定方法。
5. 統計上未觀察到差異，不等於證明兩個方法完全相同，結論須忠於檢定能力與樣本範圍。

## 論文結構與語言

1. 摘要、研究問題、方法、結果、限制與結論必須使用一致的研究範圍及術語。
2. 結果章只呈現已完成且可追溯的結果，進行中工作不提前出現在摘要、貢獻或結論。
3. 表格與圖形須在正文中說明用途、欄位、比較條件、主要發現與限制。
4. 技術名詞首次出現時提供清楚定義，後續使用相同名稱，避免同義詞任意替換。
5. 使用正式、客觀且可驗證的學術語氣，避免口語宣傳詞、代筆語氣與不必要的自我評價。
6. 繁體中文文件使用臺灣常用資訊用語，並檢查簡體字、標點、字型與圖表編號一致性。
7. 正文不包含本機路徑、下載檔名、內部審查檔名、版本管理說明或撰寫流程備註。

## 文件版本管理

1. 原始文件保持唯讀，每次修改產生新的編號版本，不直接覆寫來源。
2. 以可重現的程式或明確步驟執行修改，避免只能由人工記憶重做的操作。
3. 修改前後皆須檢查全文結構、格式、圖表、引用、欄寬與頁面配置。
4. 保留來源與輸出的雜湊值，確認修改過程未意外改動原始文件。
5. 交付前確認文件可正常開啟，內部結構完整，並重新擷取文字核對實際內容。
6. 不強制設定開檔時自動更新全部功能變數。文件若含參考文獻增益集欄位，可能因此
   出現誤導性的外部檔案警告，目錄、頁碼參照及圖表編號應保留為內部欄位並按需手動更新。

## 研究資料與可重現性

1. 保存原始資料、處理後資料、分析程式、執行條件及彙整結果，並說明彼此關係。
2. 資料清理、排除、去重與切分方式須如實記錄，不補寫不存在的程序。
3. 已知限制應與資料及方法一起保存，使後續使用者不會把描述性資料誤認為黃金標準。
4. 封存目錄應具備說明文件、機器可讀狀態及逐檔雜湊清單。
5. 公開或提交前掃描憑證、權杖、個人資料、私有主機資訊及不應發布的本機路徑。

## 交接與規則邊界

1. 通用代理規則只保存可跨專案重複使用的原則。
2. 特定研究的實驗條件、數字、進度、錯誤診斷與下一步，應保存在該研究專案的交接檔。
3. 永久規則與暫時狀態不得混寫，避免舊進度被後續工作者誤認為目前事實。
4. 交接時應指出證據位置、完成定義與未解決事項，但不得把尚未完成的內容寫成結果。
5. 若通用規則與研究專案的明確限制衝突，先遵守較嚴格且可驗證的規則，並記錄原因。
